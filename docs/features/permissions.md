# 权限体系

[← 返回 README](../../README.md)

一家之主说了算。整套体系围绕一个理念：默认什么都看不到，boss 逐人开门。

## 登录

Google 一键登录。未登录状态是一个空白网站：不记录任何数据，没有个人化内容，连栏目入口都不显示。

## 角色

| 角色 | 权限 |
|---|---|
| boss | 一切，含私密内容、用户管理、内容范围分配 |
| admin | 站点管理（后台、设置、文件巡航）、随机添加、外站条目管理 |
| regular | 只看 boss 给开的栏目 |
| guest | 空白网站 |

## 内容范围（scope）

boss 在用户管理页给每个用户勾选栏目：电影、剧集、动漫、书架、直播、体育、Missed、音乐剧、笔记。逐项勾选，或"全部开放"。

典型用法：给孩子开动漫和书架，给自己留全部。没开的栏目对该用户完全不存在：顶栏不显示入口，直接访问 URL 也会被页面和接口双重拦下。

## 私密内容

私密保险箱和旅行相册只有 boss 能看，另加一道设备级口令：新设备首次访问输一次口令，之后这台设备一年免输。

## 数据隔离

进度、收藏、笔记、历史、B站观看记录，全部按用户隔离存储。谁也看不到谁的。

## 运营可观测

- 用户行为监督后台：谁看了什么、搜了什么
- AI 成本看板：DeepSeek 每一次调用的 token 和费用，按组件（问候 / 温度 / 问答 / 注解）分账
